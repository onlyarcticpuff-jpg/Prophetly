Suggested operational flow for balance integrity and deposit tracking.
Deposit detection: backend receives provider webhook or polls blockchain API using process.env.TATUM_API, resolves the destination address to a user via profiles.deposit_wallet, then queues confirmation handling.
Confirmation credit: once the transaction is finalized, server inserts a transaction row and increments profiles.balance in one protected operation.
Withdrawal: server checks KYC/risk rules, validates amount against profiles.balance, decrypts the encrypted wallet material from profiles.wallet_key only in memory, signs via Solana web3, broadcasts, and stores the resulting signature.
Frontend responsibility: render address, show status, submit requests, and reflect server-confirmed balances only. Never hold custody secrets.
